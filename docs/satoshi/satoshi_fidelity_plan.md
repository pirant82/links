# Satoshi.html 텍스트 무결성(100% Fidelity) 및 UI 매핑 계획서

## 1. 개요
본 프로젝트의 핵심 목적은 `docs/satoshi/ko.txt`와 `docs/satoshi/en.txt`의 모든 내용을 **단 하나의 글자 유실이나 변형 없이** 이식함과 동시에, 각 텍스트의 성격에 맞는 **고퀄리티 UI 컴포넌트(이메일, 이미지 등)**를 적용하는 것이다.

## 2. 텍스트 무결성 및 디자인 원칙
- **Zero Omission**: 모든 문장, 기호, 이메일 헤더 정보를 100% 포함한다.
- **Visual Contextualization**: 단순 텍스트 나열이 아닌, 내용의 맥락에 맞는 디자인(이메일 UI, 뉴욕타임스 레이아웃 등)을 입힌다.
- **Sync Integrity**: 한국어와 영어 데이터가 동일한 컴포넌트 구조 내에서 1:1로 대응되도록 설계한다.

## 3. UI 컴포넌트 매핑 규칙 (Mapping Rules)

### 3.1. Email Interface (이메일 UI)
- **감지 패턴**: "From:", "To:", "Date:", "Subject:" (영문) 또는 "보낸 사람:", "받는 사람:", "날짜:", "제목:" (국문)
- **적용 디자인**: 빈티지 메일 클라이언트 스타일.
  - 헤더: 회색 배경 또는 상단 경계선, 굵은 폰트의 라벨 적용.
  - 본문: Monospace(고정폭) 글꼴을 사용하여 원문의 날것 그대로를 표현.
  - 포인트: 비트코인 오렌지(`--orange`) 색상을 보조 지표로 사용.

### 3.2. Article Layout (기사 레이아웃)
- **Drop Cap**: 각 섹션의 첫 번째 문단 시작 글자에 뉴욕타임스 스타일의 대문자 드롭캡 적용.
- **Subheaders**: `ko.txt`의 "I. ...", "‘...’" 등의 소제목을 감지하여 굵고 큰 Serif 폰트 제목 적용.
- **Takeaways Box**: 기사의 핵심 요약 섹션(예: "4 Takeaways")은 전용 박스 디자인 적용.

### 3.3. Image & Caption (이미지 및 캡션)
- **감지 패턴**: `(이미지: ...)` 또는 `Image...` 텍스트.
- **적용 디자인**: 
  - `docs/satoshi/original/` 폴더 내의 실제 이미지 파일 매칭.
  - 텍스트에 포함된 캡션 내용을 `figcaption` 태그로 분리하여 세련된 폰트와 여백 적용.

## 4. 데이터 구조 예시 (JSON Schema)
```json
{
  "type": "email",
  "data": {
    "header": { "from": "...", "to": "...", "date": "..." },
    "body": "원문 텍스트 100% (변경 절대 금지)"
  }
}
```

## 5. [강력 권고] 실행 지침
1. **텍스트 우선 추출**: 디자인을 입히기 전에 반드시 원문의 텍스트를 변수나 상수에 100% 먼저 확보하라.
2. **패턴 매칭 정밀도**: 이메일 헤더의 날짜 형식이나 발신자 주소 등 아주 작은 텍스트도 누락하지 마라.
3. **병렬 검증**: 언어를 전환했을 때, 이메일 UI 내부의 텍스트도 한국어/영어로 완벽하게 교체되는지 확인하라.

## 6. AI 오작동 방지 가이드
> **"디자인은 텍스트를 담는 그릇일 뿐이다. 그릇을 예쁘게 만들기 위해 내용물(텍스트)을 버리거나 깎아내지 마라."**
> - 요약 금지, 생략 금지, 변형 금지.
> - 텍스트가 길어서 코드가 복잡해지더라도, 모든 데이터를 끝까지 담아라.
